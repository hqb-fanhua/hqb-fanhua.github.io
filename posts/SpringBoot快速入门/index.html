Springboot探究

  1. pom.xml文件

    //是父项目工程
<parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.6.RELEASE</version>
        <relativePath/> <!-- lookup parent from repository -->
</parent>
//用于管理SpringBoot里的所有版本依赖，

    父工程是版本的仲裁中心，以后导入依赖是默认不用写版本（没有在父项目里管理的依赖需要版本）

  2. SpringBoot导入的依赖

     <dependency>
     <groupId>org.springframework.boot</groupId>
     <artifactId>spring-boot-starter-web</artifactId>
</dependency>

    spring-boot-starter-web:场景启动器，帮助导入 web所需要的依赖。

    starter(启动器)：SpringBoot的启动器包含了很多的坐标依赖，需要组件进行开发的时候，就可以通过导入相关的启动器来导入坐标依赖。

  3. 主程序类，主程序入口

    @SpringBootApplication
@MapperScan("com.hqb.mapper")
public class CommunityApplication {
    public static void main(String[] args) {
        SpringApplication.run(CommunityApplication.class, args);
    }
   
}

    @SpringBootApplication:SpringBoot配置类的标记，用于表明主程序的入口。

    @Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan(excludeFilters = { @Filter(type = FilterType.CUSTOM, classes = TypeExcludeFilter.class),
     @Filter(type = FilterType.CUSTOM, classes = AutoConfigurationExcludeFilter.class) })
public @interface SpringBootApplication {

    @SpringBootConfiguration：SpringBoot的配置类

    ​ 标注在某个类上，表明这是一个SpringBoot的配置类

    ​ @Configuration:配置类上标注这个注解

    ​ 配置类是容器中的一个组件

    @EnableAutoConfiguration:开启自动配置功能

    ​ SpringBoot会根据注解配置开启自动配置功能，开启后配置文件才生效

    @Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@AutoConfigurationPackage
@Import(AutoConfigurationImportSelector.class)
public @interface EnableAutoConfiguration {

    @AutoConfigurationPackage:自动配置包

    ​ @Import(AutoConfigurationImportSelector.class)

    ​ Spring的底层注解@Import,给容器中的导入了一个组件

    两个注解将主配置类（@SpringBootConfiguration标注的类）所在包及包下的所有组件扫描到容器中。

1.使用Spring Initializr 创建项目

​ ○static:主要存放静态文件,如css,images,js

​ ○template:存放模板,html页面和模板页面存放的目录

​ ○application.peoperties:进行SpringBoot相关配置的配置文件

2.SpringBoot配置文件

SpringBoot使用一个全局配置文件，配置文件名是固定的

●application.properties

●application.yml

配置文件的作用：修改SpringBoot自动配置的默认值；SpringBoot底层已经自动配置好

YAML:是一个标记语言，以数据为中心，比json,xml更适合做配置文件

3.YAML基本语法

k:(空格)​v:表示一对键值对

以空格的缩进来控制层级关系；只要是左对齐的一列数据，都是同一级的；

server:
	port:8081
	path:/hello

属性和值大小写敏感

值的写法：

​ 字面量：普通的值（数字，字符，布尔）

​ k:v直接写

​ 字符串默认不用加上单引号或者双引号

​ 双引号：会转义特殊字符

​ 单引号：不会转义特殊字符

​ 对象、Map：

​ 还是采用k:v键值对方式来写

friends:
	lastName:zhangsan
	age:20

​ 行内写法

friends{lasetName:zhangsan,age:20}

​ 数组（List、Set）

​ 用-表示数组中的一个元素

pets
	- cat
	- dog
	- pig

​ 行内写法

pets:{cat,dog,pig}
4.注解的使用

@Value获取值和@ConfigurationProperties获取值比较

@ConfigurationProperties @Value
功能 批量注入配置文件中的属性 一个个验证
松散的绑定 支持 不支持
SpEl 不支持 支持
JSR303数据校验 支持 不支持

@PropertySource:加载指定的配置文件

@Component
@PropertySource(value = {"classpath:person.properties"})
public class Person {

    private String lastName;
    private Integer age;
    private Boolean boss;
    private Date birth;

    private Map<String,Object> maps;
    private List<Object> lists;
    private  Dog dogs;
    
    //省略getter和setter
}

@ImportResource:导入Spring的配置文件，让配置文件里面的内容生效

@ImportResource(locations = {"classpath:bean.xml"})
@SpringBootApplication
public class SpringBootDemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(SpringBootDemoApplication.class, args);
    }

}

SpringBoot推荐使用全注解的方式添加组件，使用@Bean注解添加组件

配置文件的占位符

RandomValuePropertySource:配置文件中可以使用随机数

${random.uuid}

Profile:支持对不同环境提供不同的功能支持，可以通过激活、指定参数等方式快速切换环境

​ ①多profile文件形式：

​ 格式：application-{profile}.properties:

​ ●application-dev.properties、application-prod.properties

​ ②yml多文档块模式

---
server:
  port: 8081
spring:
  profiles:
    active: dev
---
server:
  port: 8082
spring:
  profiles: proud
---

​ ③激活方式

​ —命令行：spring.profiles.active = dev

​ —配置文件:spring.profiles.active = dev

​ —jvm参数:-Dspring.profiles.active = dev

5.SpringBoot配置文件加载位置

​ 默认扫描以下位置的application.properties或application.yml配置文件。

​ —file:./config/

​ —file:./

​ —classpath:/config/

​ —classpath:/

​ 以上路径是按照优先级从高到低的顺序加载，每个路径都有配置文件，会全部加载所有路径的配置文件，形 成互补配置。

​ 可以通过spring.config.location来改变文件的默认位置。
